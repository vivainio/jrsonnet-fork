= jrsonnet-fork

Personal fork of https://github.com/CertainLach/jrsonnet[CertainLach/jrsonnet].

For what jrsonnet is, how it compares to other Jsonnet implementations, build instructions, and bindings (Rust / Python / C / WASM), see the https://github.com/CertainLach/jrsonnet[upstream README].

== Difference from upstream

`--preserve-order` is exposed as an always-available, global CLI flag. In upstream it lives behind the `exp-preserve-order` compile-time feature; here it ships in every build. When set, object field insertion order is preserved through `std.manifest*`, `std.objectFields*`, and object composition.

Upstream rejected this as a global flag (https://github.com/CertainLach/jrsonnet/pull/217[#217]) because it makes `+` non-commutative on objects and silently changes `objectFields` order everywhere. That is an accepted trade-off in this fork for personal use; if you need a pure evaluator, use upstream.
